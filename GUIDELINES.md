# Coding Guidelines
This file is a description of rules and conventions that contributors to this project are expected to follow.

## Syntax conventions
- Use function template
- Use `snake_case` for variables and functions
- Keep lines to a maximum of 100 characters
- Always comment blocks of code that do one thing

## Git workflow
- Always pull from main before beginning coding work
- When working on code, make sure to create a branch and work from there
- Merge to main branch regularly
